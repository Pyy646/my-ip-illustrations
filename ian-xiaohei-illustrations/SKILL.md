# 小鱼女孩蜡笔手绘正文配图Skill

## 核心定位
为中文文章设计和生成 16:9 横版正文配图。目标不是做商业插画、PPT 信息图或甜腻卡通，而是把文章里的关键判断、流程、结构、状态或隐喻，转化成一张干净、有创意、可读性强但不生硬说明书的蜡笔手绘解释图。

默认视觉 IP：黄发小鱼发卡Q版小女孩，1.5头身，浅金长卷发+浅蓝色小鱼发卡、蓝色背带裤，断续蜡笔短线笔触，平淡软萌神态；小女孩必须参与画面核心动作，不能只站在一旁充当装饰贴纸。

## 先读这些参考
按任务需要读取，不要一次性加载全部上下文：
- `references/style-dna.md`：画风规范、配色规则、文字标注、画面禁忌
- `references/xiaohei-ip.md`：小女孩IP完整形象、神态、动作规范、禁止改动项
- `references/composition-patterns.md`：构图类型、原创隐喻设计规则、复用限制
- `references/prompt-template.md`：单张插图AI生成提示词完整模板
- `references/qa-checklist.md`：出图校验标准、迭代修改规则
- `assets/examples/`：仅作画风密度参考，禁止直接照搬案例构图、道具、标注；除非用户明确要求复刻，否则每次重新设计全新隐喻画面

## 完整工作流
### 1. 消化正文内容
先读取用户提供的正文、网页链接、Notion文档、Markdown、截图素材，提炼核心信息：
- 全文核心观点
- 存在认知转折的段落
- 适合可视化图解的内容
- 仅文字即可讲清、无需配图的内容

配图不平均分配，优先抓取「认知锚点」：核心结论、逻辑断点、输入输出闭环、流程分流、前后对比、多路径选择、踩坑预警、角色状态变化。

### 2. 输出配图策略清单
若用户需求为「分析配图思路/梳理需要配图的段落」，先输出shot list配图清单，每张图标注：
- 放置对应段落位置
- 画面主题
- 表达核心含义
- 构图结构类型
- 小女孩在画面中的核心动作
- 推荐画面元素
- 配套手写中文标注词

配图数量规范：短文1-3张；常规文章4-8张；长篇内容不超过9张，够用即可，不堆砌插图。

### 3. 单张独立生成插图
若用户明确指令「生成/出图/制作插图」，无需二次确认，调用内置`image_gen`单张分开生成，禁止多张图拼接在同一画布。

单张插图强制包含规则：
- 16:9 横向中文文章配图
- 纯白色干净背景
- 蜡笔断续短线手绘线条
- 少量红/橙/蓝手写中文标注
- 大面积留白
- 黄发小鱼发卡小女孩作为画面核心动作主体
- 禁止PPT流程图、精致商业插画、低幼甜腻卡通、复杂架构图、左上角手写标题

禁止直接复用案例构图（传送带、路径、工具箱、盖章等旧案例结构），除非用户指定复刻；每一张图都要根据当前文本设计全新独特隐喻。

### 4. 出图校验与迭代修改
生成完成后对照`references/qa-checklist.md`逐条检查，出现以下问题立即重绘或局部编辑：
- 小女孩仅作为边角装饰，未参与核心动作
- 画面元素拥挤，留白不足
- 版式死板，像标准PPT流程图
- 文字标注过多、有错别字
- 左上角出现「流程图/系统架构/常见坑」等标题文字
- 画风过于低幼、死板、甜腻
- 背景非纯净纯白色

### 5. 文件保存交付
若用户在工作空间内操作，将全部插图保存至路径：
```text
assets/<文章英文标识>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成原图，不覆盖已有历史插图，仅在用户明确要求替换时覆盖旧文件。

## 输出规范

配图策略清单简洁精炼，不冗余；交付插图时附带简短说明：

- 插图总数量
- 每张插图对应的文章用途
- 画面稳定度区分（最优插图 / 可选补充插图）

不堆砌冗长画风理论，以插图直观表达内容为主。
